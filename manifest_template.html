Manifest Template

How to Use This Template

  • This document has four types of sections:
    • Document header (“HEADER”)
    • Document footer (“FOOTER”)
    • Trip pickup (“PICKUP”)
    • Trip drop-off (“DROP OFF”)
  • Section templates are all the text between statements that are in brackets that start with “BEGIN” or “END”, followed by the name of the section type, like so: “[BEGIN HEADER]” and “[END HEADER]” (without the quotes). Each begin and end statement must be by itself on a line.
  • Except for page headers and footers, anything in this template document that is not between begin and end statements (like this how-to) will not appear on a final manifest.
  • Any trip value or associated run value can be added to an element as a field within a section. Just put the value’s column header name in braces, like so: {PU Address}
  • The document sections each appear once only — the header at the beginning of the document and the footer at the end of the document. The values used for the header will be from the trip that has the first pickup, and the values used for the footer will come from the trip that has the last drop-off.
  • You can also put values in page headers and footers. Like document headers and footers, the values used for the page header will be from the first pickup, and the values used for the page footer will come from the last drop-off. While it's possible to edit the template so that the page header and footer for the first page are different from the default page header and footer, due to limitations in Google's programming interfaces, only the default page header and footer will be applied, and they will be applied to all pages.
  • Each trip can appear twice —  once for pickup and once for drop-off. Trip pickup and drop-off elements appear in the chronological order of their respective event (pickup time for pick elements, drop-off time for drop-off elements). If the drop-off of one trip is at the same time as the pickup of another trip, the drop-off will appear first. If you prefer that each trip appear only once, then simply omit the trip drop-off element from the template.
  • Behind the scenes, Sheets stores both date and time fields the same way. To figure out whether a date/time field should be formatted as a date or as a time, the column name is used. If the column name of a date/time field contains the word “date”, then the field will be formatted as a date only. If the column name of a date/time field contains the word “time”, then the field will be formatted as a time only. If neither word is present, the field will show both the time and date.
  • The following fields that aren’t associated with any trip can also be used:
    • {Manifest Creation Date}
    • {Manifest Creation Time}
